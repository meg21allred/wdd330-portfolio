Chapter 9 The Window Object

The Browser Object Model

  • Definition - a collection of porperties and methods that contain information about the browser and computer screen.
  • Global variables - variables created without the use of const, let or var keywords, and can be accesses in all parts of the program. this variable is part of the gobal object. ex. x - 6 window.x << 6.

Finding out Browser Informations

  • Which Browser - The window object has a navigator property that returns to the Navigator object. The Navigator object contains information about the browser being used.

Browser History

  • window.history - can be used to access information about any previously visited pages in the current browser session.
  • Controlling Windows - a new window can be opened using window.open() this takes the URL of the page to be opened as its first parameter, the window title as its second parameter, and a list of attributes as the third parameter.

Cookies

  • Definition - small files that are saved locally on a user's computer as a way of getting round HTTP being a stateless protocol.
  • To set a cookie - document.cookie = 'name=Superman';
  • cookies can be given a date to expire.
  • Adding the string ; secure to the end of a cookie will ensure it's only transmitted over a secure HTTPS network.

Timing Functions

  • The window object provides some useful methods for scheduling the execution of a function, and for repeatedly executing functions at regular intervals.
  • window.setTimeout() - accepts a callback to a function as its first parameter and a number of milliseconds as its second.
  • setInterval() - same as timeout but it will repeatedly invoke the callback function after every given number of milliseconds.
  • setTimeout() and setInterval() can be used for animations, though requestAnimationFrame() is an improved way of doing the saem thing.

The "data-" attribute

  • Defintion- a way of embedding data in a web page using custom attributes that are ignored by the browser. there sole purpose is to be used by the JavaScript program.

Local Storage, Geolocation, Web Workers, and Multimedia

  • APIs help websites gain access to hardware, such as cameras, batteries, Geolocation, and graphics cards. It is best practice to use feature detection before using any of the API methods.
  • Web Storage
    • Provides key-value storage on the clients computer that is similar to cookies but has fewer restrictions.
    • Web Storage API means that the window object will have a property callse localStorage.
    • To save objects to local storage use stringify() and parse() to retrieve it.
  • Geolocation
    • Obtains the geographical position of the device being used.
  • Web Workers
    • Allow the processes to be run in the background, adding support for concurrency in JavaScript.
  • Websockets
    • Allows two-way communication with the server - aKa - push messaging.
  • Notifications
    • show messages using the system's notifications, such as a popup in the corner or the screen.
  • Multimedia
    • Audio and video tags used to insert audio and video clips into a web page.

Shims and Polyfills

  • Libraries of code that allow you to use the APIs as usual using the necessary code that is not provieded natively by the user's browser.
