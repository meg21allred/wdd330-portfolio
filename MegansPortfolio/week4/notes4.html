Chapter 8 Forms

  • Form Controls
    • It is possible to process a form on the "front end" before it is sent to the server using JavaScript
    • Parts of a form
      • Action - is the URL that the form will be submmitted to so it can be processed on the server side
      • Name - name of the form used to identy the form for interactions
  • Accessing Form Elemments
    • The forms of a document are collected in a HTML collection. "document.forms" use index notation to access froms
    • Useful Properties and methods
      • form.submit() - submit the form automatically.
      • form.reset() - reset all th form controls back to their initial values.
      • form.action() - set the action attribute of a form so it is sent to a different URL
  • Form Events
    • Focus - the focus event occurs when a element is focused on result in some sort of highlight feature chosen by the coder.
    • blur - occurs when the user moves focus away from the form element.
    • change - occurs when the user moves the focus aways from the form element after changing it.
    • Submit - submits the form
  • Submitting a form
    • JavaScript can be used to intercept the form before it is sent by adding a submit event listener.
    • preventDefault() - will prevent the default action of an event
  • Retrieving and Changing Values from a form
    • Use Value property to retrieve text inside the field - alert('You searched for: ${input.value}');
    • Value property can also be used to set the value - input.value = 'Search here';
    • You can use blur and focus to remove or place the default value when the user starts to enter their value
      • You can also achieve this by using the "placeholder" attribute
    • Form Controls
      • input - fields, including text, passwords, check boxes, radio buttons, and file uploads
      • select - menus for drop-down list of options
      • textarea - elements for longer text entry
      • buttons - elements for submitting and resetting forms
    • Newer form attributs
      • autofocus - gives focus to this element when the page loads
      • maxlength - will limit the number of characters that can be entered in the field.
  • Form validation
    • The process of checking whether a user has entered the information into a form correctly.
    • Types of form validation
      • Required - just add a required attribute to the tag
      • Using functions - add a funcition through event listener to check to see if certain requirements are met
      • preventDefault() - prevents the form from being submitted
      • Disable the submit button - add disabled attrubute to the input element

Chapter 12 Object-Oriented Programming in JavaScript

  • Constructor Functions
    • A function that defines the properties and methods of an object
    • this - is used to represent the object that will be returned by the constructor
    • Using Classes in JavaScrip
      • Class declaration syntax - class Dice {constructor(sides=6) { this.sides = sides; } }
  • Prototypes
    • A way to augment a class with extra methods and properties after it has been created.
    • The memory for a prototype property only exists in one place
    • LIVE - If a new property or method is added to the prototype, any instances of its class will inherit the
      new properties and methods automatically.
  • Public and Private methods
    • By default, an object's methods are public in JavaScript.
    • it is possible to keep some properties and methods private inside of a class declaration.
  • Inheritance
    • The Prototype Chain - a prototype is an object that has it's own prototype and so on
    • Enumerable Properties - properties of objects in JavaScript that will show up when a for - in loop is used
      to loop through an objects properties and methods.
    • Using 'extends' - the way a class can inherit from another class
    • Keyword 'super' - refers to the parent class and can be used to access any properties and call any methods of the parent class.
    • Polymorphism - different objects have the same method, but implement them in different ways.
  • Creating Objects from other objects
    • It is possible to avoid using classes altogether and create new objects based on another object that acts
      as the prototype for the new object.
  • Adding Methods to Built-in Objects
    • Monkey-Patching - the ability to add more methods to the prototype of JavaScript's built-in objects.
  • Mixins
    • A way of adding properties and methods of some objects to another object without using inheritance.
      - use Object.assign() method.
    • mixin() function for deep copies - function mixin(target, ...objects) { for(...) { loop through all the objects posessions
      and assign them to the new target.} return target;}
  • Chaining functions
    • A sequence of methods called one after the other - ex. superman.fly().move().xray()
  • This and That
    • Set a variable 'that' to equal 'this' before a nested function, and refer to that in the nested function instead of this.
  • Borrowing Methods from Prototypes
    • borrow methods brom objects without having to inherit all their properties by making a reference to the function that you
      want to borrow.

Chapter 15 Modular JavaScript

  • Modular JavaScript
    • A Module is a self-contained piece of code that provides functions and methods that can then be used in other files and by other modules
    • All code in modules is always in strict mode without the need for 'use strict' and there is no way to opt out of this.
    • A module has its own global scope, so any variables created in the the top-level of a module can only be accessed within that module.
    • The value of this in the top level of a module is undefined, rather than the global object.
    • You can't use HTML-style comments in modules.
    • import ex. import {PI} from './pi.js';
    • import all values and functions from a module ex. import * as stats from './stats.js';
