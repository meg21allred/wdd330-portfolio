Chapter 10 Testing and Debugging

  • Errors, Exceptions, and Warnings
    • Causes of problems
      • System error - there's a problem with the system or external devices with which the program is interacting.
      • Programmer error - the program contains incorrect syntax or faulty logic; or a typo.
      • User error - the user has entered data incorrectly, which the program is unable to handle.
    • Exceptions - an error that produces a return value that can then be used by the program to deal with the error.
    • Stack Trace - a sequence of functions or method calls that lead to the point where the error occurred.
    • Warnings - can occur if there's an error in the code that isn't enough to cause the program to crash.
  • The Importance of Testing and Debugging
    • A good programer should ensure that the code they write fails loudly in developement so any errors car be Identified and fixed quickly.
  • Strict Mode
    • Mode that produces more exceptions and warnings and prohibits the use of some deprecated features.
    • Writing code in strict mode can help improve its clarity and speed.
    • To use strict mode the following string must be added to the firstline of a JavaScript file: 'use strict';
    • Strict mode can be used on a "per function" basis. use the same string as the frist line in the body of the function.
    • The reccommended way to use strict mode is to place all your code into a self-invoking function.
      • This is reccommended because you can't garuntee everyone working on your project is using strict mode which with cause
        problems if you ditact all your JavaScript to be in strict mode.
    • Linting tools - ex. "JS Lint, JS Hint, and ES Lint - can be used to test the quality of JavaScript code beyond strict mode.
    • Feature Detection - The process of using an "if" statement to check whether an object or method exists before trying to call the method.
    • Debugging in the Browser - run the program with breakpoints to find bugs.
      • Options for debugging JavaScript code in the broswer:
        • The Trusty Alert - alert() stops a program from running until OK is clicked and allows the ability to effectively insert breakpoints to check the value of variables.
        • Using the Console - can be used to log the value of variables at different stages of the program.
        • Debugging Tools - a tool that allows you to set breakpoints in your code that will pause a program at certain points.
  • Error Objects
    • can be created by the host environment when an exception occurs, or it can be created in the code using a constructor function.
      ex. - const error = new Error('Opps, something when wrong');
    • Objects for Specific errors:
      • EvalError - (only for backwards compatibility) - identify errors when using the global eval() function.
      • RangeError - thrown when a number is outside an allowable range of values.
      • ReferenceError - thrown when a reference is made to an item that doesn't exist.
      • SyntaxError - thrown when there's an error in the code's syntax.
      • URIError - thrown when there's a problem encoding or decoding the URI
      • InternalError - thrown when an error occurs in the JavaScript engine.
      • TypeError - used to create custom error objects.
  • Throwing Exceptions
    • It is possible to throw your own exceptions using the "throw" statement.
    • It is best practice to throw an error object ex. - throw new Error('Something has gone wrong');
  • Exception Handling
    • Occurs when the program terminates with an error message.
    • Try, Catch, and Finally
      • Try block - will run the code inside the block as normal, but if an exception occurs it will pass the error object that is thrown onto the a 'catch' block
      • Catch block - will only run in an exception is thrown inside the 'try' block.
      • Finally block - cann be added after a catch block and will always be executed after the try or catch block, regardless of whether an exception occurred.
  • Testing Framworks
    • Provide a structure to write meaningful tests and then run them.
    • Framework Example:
      • Jest - Created by Facebook, it makes it easy to create adn run tests by providing helper methods for common test assertions.
