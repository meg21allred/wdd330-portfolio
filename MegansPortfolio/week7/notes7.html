Chapter 11 Further Functions

  • Function Properties and Methods
    • The call() method can be used to set the value of "this" inside a function to an object that is provided as the first arugment. ex. sayHello.call(clark)
    • If a function doesn't refer to an object as "this" in its body, it can still be called using the call() method, but you need to provide "null" as its first arugment.
    • Custom properties - adding your own properties to functions.
    • Memoization- a feature that provides result caching. You might want to do this if the result you are computing takes a lot of time.
      ex. square.cache; << {"3": 9, "-11" : 121 }
  • Immediately Invoked Function Expressions
    • Definition - an anonymous function that is invoked as soon as it's defined.
    • iffes are a usef way of performing a task while keeping any variables wrapped up within the scope of the function.
    • Good uses for IFFEs
      • Temporary Variables
      • Initialization Code
      • Safe Use of Strict Mode
      • Creating Self-contained Code Blocks
  • Self-defining Functions
    • Definition - a functions that is able to call itself and also define itself and even redefine itself.
    • Init-Time Branching -A technique that can be used with the feature detection to create functions that rewrite themselves.
  • Recursive Functions
    • Defintion -a function that invokes itself until a certain condtion is met.
  • Callbacks
    • Definition -functions passed to other functions as arguments and then invoked inside the
      function they are passed to.
    • Used to facilitate -Event-driven Asynchronous Programmingwhich means only one piece of code with be processed at a time.
      • used to ensure waiting for tasks to complete doesn't hold up the execution of other parts of the program.
  • Promises
    • Definition - represents the future result of an Asynchronous operation.
    • The Promise life-cycle
      • Resovled - the Asynchronous operation was completed successfully.
      • Rejected - the Asynchronous operation didn't work, didn't complete or resulted in an error.
    • A Super Promise - a pending phase while you wait on the result of an operation, then deal with the results in a proper way.
    • ex - const promise = new Promise( (resolve, reject) => { // initialization code goes here if (success) { resolve(value); } else { reject(error); } });const promise = new Promise( (resolve, reject) => { // initialization code goes here if (success) { resolve(value); } else { reject(error); } });
  • Async Functions
    • Definition - Functions preceded by the "async" keyword that allow you to write Asynchronous code as if it was synchronus.
  • Closures
    • Definition - a reference to a variable that was created inside the scope of another function, but is then kept alive and used in another part of the program
    • Generators - special functions used to produce iteratiors that maintain the state of a value.
  • Introductions to Functional Programming
    • The ability to pass functions as arguments, return them from other functions, and use anonymous functions and closures, are all fundamental elements of functional programming.
    • Pure Function features
      • Return value should only depend on the values provided as arguments.
      • There are no side effects.
      • Referential transparency.
      • Must have at least one argument.
      • Must have a return value.
  • Currying
    • Definition - a process that involves the partial application of functions.

Chapter 13 Ajax

  • Communicating with the Server Using Fetch API
    • Fetch API - a living standard for requesting and sending data asynchronously across a network.
    • JSON Responses - the method used to deal with transforming data into a promise thah resolves to a JavaScript object
    • Request InterFace - Request object that allows a numver of options to ve set about the request.
      • Request() constructor
        • url - The URL of the requested resource.
        • method - a string that specifies which HTTP method should be used for the request. ('GET' or 'POST')
        • headers - provides details of the request's headers
        • mode - allows you to specify if CORS is used or not.
        • cache - allows you to specify how the request will use the browser's cache.
        • credentiols - lets you specify if cookies should be allowed with the request.
        • redirect - specifies what to do if the response returns a redirect.
  • Form Data
    • If a form is passed to the FormData() constructor function as an argument, the form data instance will serialize all the data automatically, ready to be sent using Ajax.
